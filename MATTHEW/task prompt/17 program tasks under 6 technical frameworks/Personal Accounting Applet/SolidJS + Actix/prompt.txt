Build a personal finance manager using emerging technologies:

**Technology Stack:**
- Frontend: SolidJS with TypeScript
- Backend: Actix Web
- ORM: SeaORM
- Database: PostgreSQL (Docker)

**Database:**
Connect to existing PostgreSQL with tables:
- transactions (id, amount, type, category, description, transaction_date, created_at)
- categories (id, name, type)

**Core Features:**
1. Add income/expense transactions
2. View transactions with month filter
3. Monthly summary dashboard
4. Category spending visualization

**API Endpoints:**
- GET /api/transactions?month=2025-08
- POST /api/transactions
- GET /api/summary?month=2025-08
- GET /api/categories

**Frontend Requirements:**
- Use SolidJS with TypeScript
- Implement client-side routing
- Use minimalist CSS framework
- Include form validation

**Backend Requirements:**
- Use Actix Web framework
- Implement SeaORM for database operations
- Add proper error handling with custom error types
- Include request validation

**Database Connection:**
- Database::connect("postgres://postgres:financepass@localhost:5432/personal_finance").await?;

Generate the complete application with focus on performance and memory safety.
